# Монгол дугаар таних — Датасет ба моделийн судалгаа

Энэ баримт нь Roboflow Universe болон бусад нээлттэх эх сурвалжаас Монгол улсын
автомашины дугаар таних (ALPR/ANPR) зорилгоор ашиглаж болох датасет, бэлэн модель,
кодын сангуудыг судалсан үр дүн юм. Системийн дугаар таних урсгал нь гурван шаттай
(машин илрүүлэх → дугаарын байрлал олох → текст унших) тул эх сурвалж бүрийг аль
шатанд тохирохоор нь үнэлэв.

## 1. Хамгийн чухал дүгнэлт

Roboflow Universe болон GitHub дээр **Монгол дугаарт тусгайлан сургасан, шууд татаж
ашиглах боломжтой бэлэн YOLOv8 `.pt` жин (weights) одоогоор алга**. Гэвч дугаарын
**байрлал олох (plate detection) шатанд** шууд сургаж болох чанартай датасет байгаа
тул хамгийн бодит зам бол энэ датасетийг YOLOv8-аар сургаж өөрсдийн `mn_plate_yolov8.pt`
жинг гаргаж, системийн `PLATE_MODEL_PATH`-д залгах явдал юм.

## 3. GitHub эх сурвалжууд

**`gumbernator/Mongolian-ALPR`** (https://github.com/gumbernator/Mongolian-ALPR) нь
600+ Монгол машины зураг дээр хийсэн хоёр шаттай ALPR төсөл (Apache-2.0). Гэвч
README дээр **жингийн (model weights) холбоос алдагдсан** гэж бичсэн тул одоогоор
шууд ажиллуулах боломжгүй. Арга барил (TensorFlow OD API + дүрс боловсруулалт + CNN
тэмдэгт таних) болон Монгол дугаарын онцлогийг ойлгоход хэрэгтэй лавлагаа болно.
Энэ нь Монгол дугаарын байршил тогтоогч (location identifier) үсэг олон давтагддаг
учир тэмдэгтийн тэнцвэргүй байдал үүсдэг гэх чухал ажиглалтыг тэмдэглэсэн.

Ерөнхий (Монгол биш) лавлагаа болохуйц төслүүд: `Muhammad-Zeerak-Khan/Automatic-
License-Plate-Recognition-using-YOLOv8`, `ahasera/alpr-YOLOv8-YOLOv4Tiny` — эдгээр
нь YOLOv8 + OCR pipeline-ийн жишээ код, ерөнхий plate weights агуулдаг.

## 4. Hugging Face эх сурвалжууд

Монгол дугаарт тусгайлсан ALPR модель Hugging Face дээр олдсонгүй. Гэвч хэрэг болж
магадгүй ерөнхий загварууд: `autolane/rfdetr-alpr` (RF-DETR дугаар детектор, edge-д
тохирсон), `basolw/fast-alpr` цуглуулга. Монгол текст боловсруулалтын хувьд
`bayartsogt/albert-mongolian` зэрэг хэл боловсруулалтын модель байгаа боловч эдгээр
нь дугаар таних дүрс боловсруулалтад шууд хамаарахгүй.

## 5. Монгол дугаарын формат (OCR тохиргоонд чухал)

Монгол улсын энгийн автомашины дугаар нь **4 цифр + 3 кирилл үсэг** (жишээ:
`1234 УБА`), цагаан дэвсгэр дээр хар тэмдэгттэй. Эхний эсвэл сүүлийн үсэг нь аймаг/
нийслэлийг заадаг. 2018 оноос Roboto/DIN 1451 фонт хэрэглэж эхэлсэн. Энэ нь манай
системийн **OCR allow-list**-ийг кирилл үсэг (А-Я) + тоо (0-9)-оор тохируулах ёстойг
харуулна; зөвхөн латин үсгээр тохируулбал алдаа гарна.

## 6. Эцсийн зөвлөмж ба алхамууд

Богино хугацаанд хамгийн бодит шийдэл бол **`mongolian-plate` датасетийг (984 зураг)
Roboflow-оос YOLOv8 форматаар татаж аваад дугаарын байрлал олох детектор сургаж**,
гарсан `best.pt`-г төслийн `models/mn_plate_yolov8.pt` болгон хуулж, `.env`-д
`PLATE_MODEL_PATH=/app/models/mn_plate_yolov8.pt` гэж зааж өгөх явдал юм. Текст
унших шатанд эхлээд `OCR_LANGUAGES=mn,en` болон кирилл allow-list-ээр EasyOCR-ийг
тохируулж туршина. Хэрэв EasyOCR-ийн нарийвчлал хүрэлцэхгүй бол ирээдүйд тусгай
тэмдэгт таних (recognition) модель сургах шаардлагатай — энэ нь илүү их өгөгдөл (Монгол
дугаарын тэмдэгт бүрийн annotation) шаардана.

Дараагийн алхамд би хүсвэл (а) Roboflow-оос датасет татаж YOLOv8 сургах бэлэн
`train.py` скрипт бэлдэх, эсвэл (б) сургасан жинг яаж төсөлд залгахыг алхам алхмаар
харуулах боломжтой.
